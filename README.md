# Rdio Scanner — AkumasCoffin fork

This is a fork of [chuot/rdio-scanner](https://github.com/chuot/rdio-scanner) with extra
features layered on top of the upstream base: call transcription via Groq Whisper, a
public REST API for downstream consumers, share-link deep linking on the search page,
a Postgres-aware stats page that renders in the viewer's local timezone, and a native
Android client. All upstream functionality is preserved — recorders that worked with
the original still work here.

Original project, credit, and documentation:
**[https://github.com/chuot/rdio-scanner](https://github.com/chuot/rdio-scanner)**.

## What is it ?

[Rdio Scanner](https://github.com/chuot/rdio-scanner) is open-source software that
ingests and distributes audio files generated by software-defined radio recorders.
The web interface reproduces the experience of a real police scanner. You listen to
calls on any modern browser using the bundled web app, or via the native Android app
in this fork.

## Recorders compatibility

Rdio Scanner works with any radio recorder that can write audio files separated by
conversations or transmissions:

| Recorder                                                       | API | Dirwatch |
| -------------------------------------------------------------- | --- | -------- |
| [Trunk Recorder](https://github.com/robotastic/trunk-recorder) | X   | X        |
| [RTLSDR-Airband](https://github.com/szpajder/RTLSDR-Airband)   |     | X        |
| [SDRTrunk](https://github.com/DSheirer/sdrtrunk)               |     | X        |
| [voxcall](https://github.com/aaknitt/voxcall)                  | X   |          |
| [ProScan](https://www.proscan.org/)                            |     | X        |
| [DSDPlus Fast Lane](https://www.dsdplus.com/)                  |     | X        |
| DSD FME (with custom metadata mask)                            |     | X        |

# Install — pre-built release (recommended)

Pre-built binaries embed the web app, so there's nothing else to install. The fork
publishes Linux and Windows builds on the [releases page](https://github.com/AkumasCoffin/rdio-scanner/releases).

1. Grab the binary for your platform from the [latest release](https://github.com/AkumasCoffin/rdio-scanner/releases/latest):

   | Operating system | Architecture | File                                          |
   | ---------------- | ------------ | --------------------------------------------- |
   | Linux            | amd64        | `rdio-scanner-linux-amd64-vX.Y.Z`             |
   | Windows          | amd64        | `rdio-scanner-windows-amd64-vX.Y.Z.exe`       |

   (Other targets like FreeBSD, macOS, ARM are not pre-built for this fork — see
   [Manual install / build from source](#manual-install--build-from-source).)

2. Drop the binary somewhere on disk, e.g.:

   ```bash
   # Linux
   sudo install -d /opt/rdio-scanner
   sudo mv rdio-scanner-linux-amd64-vX.Y.Z /opt/rdio-scanner/rdio-scanner
   sudo chmod +x /opt/rdio-scanner/rdio-scanner
   ```

3. First-run setup — point it at your database, save the config, then run:

   ```bash
   cd /opt/rdio-scanner

   # Postgres (recommended for any non-trivial install)
   ./rdio-scanner --db_type postgres \
       --db_host  localhost \
       --db_port  5432 \
       --db_user  rdio \
       --db_pass  'CHANGEME' \
       --db_name  rdio_scanner \
       --listen   :3000 \
       --config_save

   # Or SQLite (zero-config, single-file DB, fine for small deployments)
   ./rdio-scanner --listen :3000 --config_save
   ```

   `--config_save` writes the flags to `rdio-scanner.ini` next to the binary. On
   subsequent runs you can just invoke `./rdio-scanner` with no args.

4. Open `http://<your-host>:3000`. Admin UI lives at `/admin`. First-time login is
   **`rdio-scanner`** — it'll prompt you to change it on the first save.

5. To run as a service, see [systemd setup](#run-as-a-systemd-service-linux).

## Run as a systemd service (Linux)

Create `/etc/systemd/system/rdio-scanner.service`:

```ini
[Unit]
Description=Rdio Scanner
After=network.target postgresql.service

[Service]
Type=simple
User=rdio
WorkingDirectory=/opt/rdio-scanner
ExecStart=/opt/rdio-scanner/rdio-scanner
Restart=on-failure
# Optional: set the timezone for logs (does NOT affect stats — those render
# in the viewer's browser timezone since v6.8.0).
Environment=TZ=America/New_York

[Install]
WantedBy=multi-user.target
```

Then:

```bash
sudo useradd -r -s /bin/false rdio
sudo chown -R rdio:rdio /opt/rdio-scanner
sudo systemctl daemon-reload
sudo systemctl enable --now rdio-scanner
journalctl -u rdio-scanner -f      # follow logs
```

# Manual install / build from source

The repo is split into two builds — the Angular web app, and the Go server. The Go
server embeds the web app's compiled assets at build time, so the final artifact is
a single binary.

## Prerequisites

- **Go** 1.21 or newer ([install](https://go.dev/dl/))
- **Node.js** 18 or newer + **npm** ([install](https://nodejs.org/))
- **Git**
- For the Android client only: **Android Studio** with SDK 35 + JDK 17

## Build

```bash
git clone https://github.com/AkumasCoffin/rdio-scanner.git
cd rdio-scanner

# 1) Build the Angular web app (compiled into ../server/webapp/)
cd client
npm ci
npx ng build --configuration=production
cd ..

# 2) Build the Go server (which embeds the webapp via //go:embed)
cd server
go build -o rdio-scanner .

# 2a) Or cross-compile for a different target:
GOOS=linux   GOARCH=amd64 go build -o rdio-scanner-linux-amd64 .
GOOS=windows GOARCH=amd64 go build -o rdio-scanner-windows-amd64.exe .
GOOS=darwin  GOARCH=arm64 go build -o rdio-scanner-macos-arm64 .
GOOS=linux   GOARCH=arm64 go build -o rdio-scanner-linux-arm64 .

# The resulting binary is fully self-contained — copy it to your target host.
./rdio-scanner --help
```

## Android client (optional)

```bash
cd android
./gradlew assembleRelease
# APK output: android/app/build/outputs/apk/release/rdio-X.Y.Z.apk
# Sideload onto your device. The fork's Android app talks to the same
# server endpoints as the web app, no extra setup required.
```

# Database — PostgreSQL setup

SQLite works out of the box (single file, no setup), but Postgres is recommended
for any non-trivial install — it survives concurrent ingest + transcription +
search, scales to millions of calls, and unlocks the trigram + BRIN indexes the
fork adds for fast transcript and date-range queries.

## 1. Install Postgres

```bash
# Ubuntu / Debian
sudo apt update
sudo apt install postgresql

# Or use the official repo for a newer version (e.g. Postgres 16):
sudo install -d /usr/share/postgresql-common/pgdg
sudo curl -fsSL -o /usr/share/postgresql-common/pgdg/apt.postgresql.org.asc \
    https://www.postgresql.org/media/keys/ACCC4CF8.asc
echo "deb [signed-by=/usr/share/postgresql-common/pgdg/apt.postgresql.org.asc] \
    https://apt.postgresql.org/pub/repos/apt $(. /etc/os-release && echo $VERSION_CODENAME)-pgdg main" \
    | sudo tee /etc/apt/sources.list.d/pgdg.list
sudo apt update
sudo apt install postgresql-16

sudo systemctl enable --now postgresql
```

## 2. Create a database and user

```bash
sudo -u postgres psql <<'SQL'
CREATE USER rdio WITH PASSWORD 'CHANGEME';
CREATE DATABASE rdio_scanner OWNER rdio;

-- Enable pg_trgm so the fork's trigram GIN index on transcripts can be
-- created. If your rdio role doesn't have CREATE on the database, install
-- the extension as superuser here — the migration is idempotent and will
-- pick it up automatically on next startup.
\c rdio_scanner
CREATE EXTENSION IF NOT EXISTS pg_trgm;
GRANT CREATE ON DATABASE rdio_scanner TO rdio;
SQL
```

## 3. Allow rdio-scanner to reach Postgres

By default Postgres only listens on `localhost`. If rdio-scanner runs on the
same machine, you're done. Otherwise edit `pg_hba.conf` (typically at
`/etc/postgresql/16/main/pg_hba.conf`) and add an entry like:

```
# TYPE  DATABASE        USER  ADDRESS          METHOD
host    rdio_scanner    rdio  192.168.1.0/24   scram-sha-256
```

And `postgresql.conf` to listen on the right interface:

```
listen_addresses = '*'
```

Then `sudo systemctl restart postgresql`.

## 4. Point rdio-scanner at it

```bash
./rdio-scanner --db_type postgres \
    --db_host localhost --db_port 5432 \
    --db_user rdio --db_pass 'CHANGEME' \
    --db_name rdio_scanner \
    --config_save
```

On first start, rdio-scanner runs all schema migrations automatically. Watch the
log for `running database migration 20260424100000-dateTime-brin-idx` etc. — these
add the BRIN index on `dateTime` and the pg_trgm GIN index on `transcript`, both
of which materially speed up search and stats on large tables.

## 5. (Optional) Migrate from an existing database

If you're moving from a SQLite or another Postgres host, the simplest approach is
`pg_dump` + `pg_restore` between two Postgres instances:

```bash
pg_dump --format=custom --no-owner --no-privileges "$OLD_DSN" |
    pg_restore --no-owner --no-privileges --clean --if-exists -d "$NEW_DSN"
```

For Postgres-to-Postgres migration with optional skip-the-calls-table support, the
fork includes a helper script as a starting point at `scripts/migrate-db.sh` in
the repo (you may need to pull it from history; it's gitignored from the tracked
tree because it's deployment-specific).

# Configuration reference

All flags can be persisted to `rdio-scanner.ini` with `--config_save`. Reference:

| Flag                  | Default                | Notes                                        |
| --------------------- | ---------------------- | -------------------------------------------- |
| `--listen`            | `:3000`                | HTTP listen address                          |
| `--config`            | `rdio-scanner.ini`     | Config file path                             |
| `--db_type`           | `sqlite`               | `sqlite`, `mariadb`, `mysql`, or `postgres`  |
| `--db_file`           | `rdio-scanner.db`      | SQLite file path (sqlite only)               |
| `--db_host`           | `127.0.0.1`            | DB host (mysql/mariadb/postgres)             |
| `--db_port`           | `5432` / `3306`        | DB port                                      |
| `--db_user`           |                        | DB user                                      |
| `--db_pass`           |                        | DB password                                  |
| `--db_name`           |                        | DB name                                      |
| `--admin_password`    |                        | Set the admin password and exit              |
| `--version`           |                        | Print version and exit                       |
| `--service install`   |                        | Install as a system service                  |

Most of the runtime configuration (systems, talkgroups, API keys, downstream
endpoints, transcription settings, etc.) is set via the web admin UI at
`/admin` after the server is running.

# Fork-specific features

Beyond what upstream chuot/rdio-scanner provides, this fork adds:

- **Call transcription** — Groq Whisper with multi-key round-robin, per-system
  prompts, per-talkgroup gating, automatic retry on the next key for 429/5xx.
- **Public REST API** at `/api/v1/calls{,/:id,/transcript,/audio}` with API-key
  auth (Bearer / `X-API-Key` / `?key=`) for downstream tooling.
- **Share-link deep linking** on the search page — every call row has a copy
  link; opening `?call=<id>` opens search, anchors the date filter, highlights
  the row, and starts playback.
- **Stats page** with bucketing in the viewer's browser timezone (single shared
  cache, no `?tz=` plumbing, no server-side TZ awareness — wire is pure UTC).
- **Performance**: pg_trgm GIN on transcript, BRIN on dateTime, HTTP gzip
  middleware, WebSocket `permessage-deflate`, early-WS opener in `index.html`,
  hashed-asset `Cache-Control: immutable`.
- **LCD UI**: persistent transcript box, LED-color-coded styling, 15-row history,
  transcript poll fallback.
- **Native Android client** (`android/` directory) with multi-profile support.
- **Various reliability fixes**: audio decode generation counter, config save
  wrapped in single Postgres tx, instant listener count push, deep-link audio
  autoplay-on-gesture, etc.

See the [release notes](https://github.com/AkumasCoffin/rdio-scanner/releases) for
the full per-version breakdown.
